# Project Spec: WaveTutor (Sound Speed Simulator)

## 1. Core Objectives
WaveTutor is an interactive educational simulator for Ultrasound Nondestructive Testing (NDT) to measure longitudinal and shear sound speeds ($V_L$ and $V_S$) using the through-transmission method on a metal sample.

### Student Inputs (via UI):
1. **Material Selection**: One of 9 metals (Lead, Iron, Aluminum, Molybdenum, Magnesium, Tin, Niobium, Nickel, Zinc).
2. **Sample Thickness ($d$)**: User-defined thickness in millimeters (e.g., $20\text{ mm}$ or $25\text{ mm}$).
3. **Transducer Frequency ($f_0$)**: Selectable between $500\text{ kHz}$ and $1000\text{ kHz}$.

---

## 2. Generated Signals (Inputs to the Agents)
The system programmatically generates two text files (`.txt` format with `Time_us` and `Amplitude` columns) containing the signals:

1. **Excitation Signal (`[material]_excit.txt`)**: 
   * A $5$-cycle Tukey burst centered at the selected frequency ($f_0$).
   * Background random noise.
   * Optional startup artifact at $t=0$.

2. **Received Longitudinal Signal (`[material]_longi.txt`)**:
   * **Family 1 (Trap)**: Beginning startup artifact at $t=0$ (simulating electrical cross-talk/noise).
   * **Family 2 (True)**: True acoustic longitudinal wave arriving at $t_L = d / V_L$.
   * Background random noise.

3. **Received Shear Signal (`[material]_shear.txt`)**:
   * **Family 1 (Trap 1)**: Beginning startup artifact at $t=0$.
   * **Family 2 (Trap 2)**: An early arrival peak cluster near $t_L$ (simulating mode conversion or early shear noise).
   * **Family 3 (True)**: True acoustic shear wave arriving at $t_S = d / V_S$.
   * Background random noise.

---

## 3. Signal Processing Utility
* **Cross-Correlation Tool**: A Python function that computes the cross-correlation between the excitation signal and the received signals to detect the peak matching positions (time delays $\Delta t$).
* This outputs a list of peak delay times (e.g., Peak 1, Peak 2, Peak 3).

---

## 4. Multi-Agent Feedback Loop

```mermaid
sequenceDiagram
    autonumber
    actor Student
    participant UI as Streamlit UI
    participant Generator as Signal Generator (Python)
    participant Solver as Solver Agent
    participant Guardrail as Guardrail Agent
    participant Teacher as Teaching Agent

    Student->>UI: Select Metal, Thickness, & Frequency
    UI->>Generator: Run generator script
    Generator->>UI: Save wave files & plot graphs
    Student->>UI: Click "Run Agent Tutor"
    UI->>Solver: Send wave files & parameters

    Note over Solver: Attempt 1: Picks first peak (Trap 1: t=0 noise)
    Solver->>Guardrail: Send calculated sound speeds
    Note over Guardrail: Verifies rules against metal database limits
    Guardrail->>Guardrail: Flags Category 2 Error (Too fast / t=0)
    Guardrail-->>Solver: Reject. Return Category 2 Error log.

    Note over Solver: Attempt 2: Moves to second peak (Longi=correct; Shear=Trap 2)
    Solver->>Guardrail: Send updated calculations
    Note over Guardrail: Flags Category 1 Error (Shear vs Longi speed overlap)
    Guardrail-->>Solver: Reject. Return Category 1 Error log.

    Note over Solver: Attempt 3: Moves to third peak (Shear=correct)
    Solver->>Guardrail: Send final calculations
    Note over Guardrail: Verifies both speeds match bounds
    Guardrail->>Guardrail: Logs PASS

    Guardrail->>Teacher: Send validation logs & error history
    Solver->>Teacher: Send peak traces & math steps
    Note over Teacher: Reads lab reference docs & builds explanation
    Teacher->>UI: Render Socratic step-by-step tutorial
```

### Agent Roles:
* **Solver Agent**: Programmed to behave like a student. It calculates velocities but starts by selecting the first peak clusters (the artifacts/noise) to trigger learning moments. It recalibrates step-by-step upon Guardrail rejection.
* **Guardrail Agent**: Evaluates the Solver's calculations against the 9-metal velocity database bounds. It logs **Category 2 Errors** (impossibly fast speed from $t=0$ peaks) and **Category 1 Errors** (shear-longitudinal velocity mismatch).
* **Teaching Agent**: Collects the feedback loop history and prints the tutorial showing the student why the Solver got tricked, why the Guardrail rejected it, and how to find the correct peaks.

---

## 5. Custom Agent Skills (`.agents/skills/`)
1. **`sound-speed-calculator`**: Formulas and math/unit conversions.
2. **`signal-analyzer`**: Instructions to run cross-correlation and handle peak sequences.
3. **`guardrail-validator`**: Checks against the metal velocity bounds database.
4. **`socratic-tutor`**: Tutoring methods (e.g. Socratic style, common pitfalls) using reference guides.

---

## 6. Grounding Resources (`references/`)
Text documents describing lab physics:
* `references/cross_correlation_ndt.txt`: Explanation of how correlation locates waves.
* `references/transducer_noise.txt`: Details on electrical cross-talk (initial bang) at $t=0$.
* `references/wave_modes.txt`: Physics of longitudinal vs. shear waves and why their speeds differ.

---

## 7. Streamlit User Interface
* **Sidebar**: Inputs for Metal, Thickness, and Transducer Frequency.
* **Main Screen**: 
  * Live plots of the generated excitation and received signals.
  * Interactive text fields for the student to guess the sound speeds.
  * A "Run Agent Tutor" button to trigger the multi-agent debug-and-learn process.
